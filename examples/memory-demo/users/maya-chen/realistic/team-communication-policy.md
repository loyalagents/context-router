# Platform Engineering Communication Policy

Owner: Maya Chen

This document covers working agreements for the Signal Harbor platform
engineering team. It is not a company-wide policy.

## Day-To-Day Updates

Use Slack for quick coordination, blockers, review nudges, and operational
questions. Maya prefers Slack for same-day team communication because it keeps
threads visible to the group.

Use email for weekly summaries, longer decisions, or external stakeholder
updates. Maya prefers email when the message should be easy to forward or find
later.

Avoid duplicating the same update in every channel unless there is an incident
or a deadline-sensitive launch.

## Meetings

Default meeting length is 25 minutes. Include an agenda when asking for more
than 25 minutes. Decisions should be captured in the project note, not only in
chat.

## Escalation

For production incidents, page the on-call rotation first. After paging,
announce the incident in the platform incident channel with impact, owner, and
current mitigation.

## Noise To Ignore For Memory Demos

- The data warehouse team uses a different escalation path.
- Customer support runs a separate weekly summary process.
- Contractor access is reviewed every quarter.
